# Gemini API 비용 분석

## 사용 모델

- **모델**: `gemini-2.5-flash`
- **Thinking**: 비활성 (`thinkingBudget: 0`)
- **출력 형식**: JSON (`responseMimeType: 'application/json'`)
- **최대 출력 토큰**: 4,096

## 가격표 (유료 1등급 기준)

| 항목 | 가격 (per 1M tokens) |
|------|---------------------|
| 입력 (텍스트/이미지) | $0.30 |
| 출력 (텍스트) | $2.50 |
| 컨텍스트 캐싱 입력 | $0.03 |

> 출처: https://ai.google.dev/gemini-api/docs/pricing

## 토큰 계산 방식

### 텍스트

- 약 4글자 = 1토큰
- 한국어는 영어보다 토큰 효율이 낮음 (대략 2~3글자 = 1토큰)

### 이미지

- **768×768 픽셀 타일** 단위로 분할
- **타일당 258 토큰** 고정
- 384px 이하 이미지: 258 토큰 (1타일)

| 이미지 해상도 | 타일 수 계산 | 토큰 |
|-------------|-----------|------|
| 800×1200 | ceil(800/768) × ceil(1200/768) = 1×2 = 2 | 516 |
| 1240×1754 (A4 150dpi) | 2×3 = 6 | 1,548 |
| 1654×2339 (A4 200dpi) | 3×4 = 12 | 3,096 |
| 2480×3508 (A4 300dpi) | 4×5 = 20 | 5,160 |

> 출처: https://ai.google.dev/gemini-api/docs/tokens

## API 호출 구조

서비스에는 2개의 Supabase Edge Function이 있음:

### 1. `extract-answer-structure` (정답지 분석)

- **입력**: 정답지 이미지 + 시스템 프롬프트 (~1,200자 한국어)
- **출력**: JSON (문제번호, 정답, 지문 텍스트)

### 2. `extract-exam-structure` (학생 답안 분석)

- **입력**: 학생 시험지 이미지 + 시스템 프롬프트 (~700자 한국어)
- **출력**: JSON (학생명, 문제번호, 답안)

## 비용 절감 팁

1. **이미지 해상도 최적화**: 150~200dpi면 충분. 300dpi 이상은 비용만 증가
2. **컨텍스트 캐싱 활용**: 동일 정답지를 반복 분석할 경우 캐싱 사용 시 입력 비용 90% 절감
3. **`gemini-2.5-flash-lite`로 전환 시**:
   - 입력: $0.10/1M (67% 절감)
   - 출력: $0.40/1M (84% 절감)
   - 호출당 약 $0.001 수준으로 감소
4. **`thinkingBudget: 0` 유지**: thinking 토큰은 별도 과금되므로 비활성 상태 유지 권장
